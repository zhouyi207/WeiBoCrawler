# YssCrawler

本项目核心是做一个媒体信息的爬虫系统，比如说微博，抖音，快手，小红书，贴吧，知乎，将采集到的信息格式化存储供学术交流使用

## 项目架构

使用 CQRS 架构，做一个消息队列，在前端发送爬取请求给后端，后端创建消息并存储在消息队列中，使用 tauri 的 channel 通道更新前端的消息队列的进度

## 前端

前端展示：内容由 Sidebar 进行全局导航管理，整体分为五个一级页面模块：主页、采集页、账号页、IP页、数据库页。

---

## 主页

主要用于系统整体运行状态展示，包括：

* 任务运行总览（运行中 / 等待中 / 已失败）
* 各平台采集状态汇总
* 账号健康状态概览
* IP 可用性与风控状态概览
* 最近任务日志与异常提示

---

## 采集页

采集页采用“平台 + 任务”的分层结构展示，按平台进行分组管理，各平台下包含多个采集任务。

### 平台分组（下拉结构）

* 微博
* 抖音
* 快手
* 小红书
* 贴吧
* 知乎

### 每个平台内部包含采集任务类型

* 关键词采集任务
* 用户主页采集任务
* 热点/榜单采集任务（视平台能力扩展）

### 每个采集任务包含

* 任务配置（关键词 / 用户列表 / 规则）
* 账号池绑定
* IP池绑定
* 采集策略（轮询 / 随机）
* 速率限制配置
* 运行状态（运行 / 暂停 / 异常）

---

## 账号页

账号页按平台进行分类管理，用于统一维护各平台账号资源。

### 平台分组

* 微博账号
* 抖音账号
* 快手账号
* 小红书账号
* 贴吧账号
* 知乎账号

### 每个账号包含

* 登录状态
* 当前绑定 IP
* 历史 IP 使用记录
* 风控状态（正常 / 限制 / 异常）
* cookie / token 状态
* 最近活跃时间
* 可分配状态（可用 / 占用 / 冻结）

---

## IP页

IP页用于代理资源管理与调度状态监控。

### IP列表信息

* IP 地址
* 代理类型（HTTP / SOCKS5）
* 当前绑定账号数量
* 当前运行任务数量
* 可用状态（可用 / 限制 / 失效）
* 风控评分
* 响应延迟

### IP调度信息

* 当前分配策略（轮询 / 权重 / 绑定优先）
* 使用历史记录
* 黑名单状态

---

## 数据库页

用于采集数据的统一存储与查询管理。

### 数据分类

* 按平台分类数据（微博 / 抖音 / 等）
* 按任务分类数据
* 按关键词 / 用户维度索引

### 功能

* 数据表格展示
* 条件筛选（时间 / 平台 / 任务）
* 数据导出（JSON / CSV）
* 数据清洗状态标记
* 去重与重复记录管理

---

## 全局设计原则

* 所有资源（账号 / IP / 任务）均必须可追踪、可绑定、可审计
* 所有采集行为必须显式绑定：平台 + 任务 + 账号 + IP
* UI 结构采用“平台优先 + 资源解耦”的管理模式
* 采集执行与资源管理解耦，避免强耦合导致调度混乱
